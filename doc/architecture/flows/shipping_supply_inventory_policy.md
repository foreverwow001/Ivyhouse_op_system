# 出貨用品人工盤點與調整規則

更新日期：2026-04-03

Authoritative source：是

## 目的

本文件定義 Ivyhouse OP System 對出貨用品 / 行政耗材中「人工盤點型」項目的流程基線。這些用品必須進 ERP 管理，但本版明確採用「不在出貨時自動扣除、改以人工盤點與調整維護庫存」的營運規則。

## 適用範圍

- 紙箱
- 氣泡紙
- 膠帶
- 其他直接用於包裝、出貨保護或交付支援的耗材

本文件不適用於：

- 生產耗料原料
- 配方中的正式 BOM 項目
- 已透過轉換扣帳規則、組成規則或正式包裝規則管理的內外包材主檔項目
- 雖屬 `出貨及行政耗材` 類別，但已被定義為正式入表、入扣帳的項目，例如禮盒姓名貼

## 核心原則

### 1. 用品必須先成為正式主資料

- 出貨用品 / 包裝耗材必須有正式主資料，不得只以人工盤點表或採購備註管理。
- 每項用品至少要有正式代碼、名稱、類型、採購單位、庫存單位、狀態與停用規則。

### 2. 不在出貨時自動扣除

- 出貨建立、揀貨、包裝與交付流程，不直接觸發這類用品的自動扣帳。
- 出貨用品不得被誤建成銷售商品組成自動扣除項或配方耗料項。
- 若某項物料會作為內包裝包材或正式包裝組成被追蹤耗用，它就不屬本文件範圍，應回到包材主檔與正式扣帳規則治理。

### 3. 庫存透過人工盤點與調整維護

- 這類用品的數量變化，透過盤點事件、差異調整事件或必要的手動耗用調整維護。
- 每次盤點或調整都必須保留操作者、時間、原因與數量差異。

## 建議流程基線

1. 採購或補貨入庫後，用品進入正式庫存。
2. 日常出貨使用不自動扣帳，只累積實體耗用。
3. 依盤點週期或例外需求進行人工盤點。
4. 盤點後以調整事件更新庫存台帳，並記錄差異原因。

## 盤點頻率與提醒

- 系統初期：每週至少一次人工盤點。
- 系統穩定後：每月至少一次人工盤點。
- 若超過一個月未完成盤點，系統應顯示提醒，不得把這類用品視為可無限期不盤點的非正式項目。
- 若遇大量補貨、倉內搬動、品項切換或例外耗損，應允許在週期外加開盤點 / 調整。

## 調整原因字典基線

人工盤點型出貨用品至少應支援以下原因分類：

- `盤點差異`：例行盤點發現帳實不符。
- `破損 / 報廢`：用品損壞、污染或無法繼續使用。
- `遺失 / 盤虧`：實物短少但無對應正式耗用事件。
- `補貨建檔修正`：補登入庫、單位修正或初始化建檔補正。
- `規格切換停用`：用品改版、替換或停用導致舊規格結清。

若後續營運需要更細緻原因字典，應透過正式治理擴充，不得由盤點人員自由輸入成新的正式分類。

## Phase 1 角色邊界

- 在正式倉管 / 履約 RBAC 擴版前，人工盤點型出貨用品沿用 inventory-count Phase 1 過渡角色邊界：`生產`、`包裝`、`會計`、`主管` 可建立與完成一般盤點。
- opening balance 或中斷補救若需取消當前 session，仍只允許 `主管` 執行，且必須提供取消原因。
- `管理員` 不得因系統維運身分而直接放寬這類高風險盤點 / 補正操作的業務 approver 邊界。

## 狀態與資料要求

- 主資料狀態仍採 `草稿`、`啟用`、`停用`。
- 若用品停用，必須保留歷史盤點與調整追溯，不得直接刪除。
- 建議最少要有以下欄位：
  - `用品代碼`
  - `用品名稱`
  - `用品類型`
  - `採購單位`
  - `庫存單位`
  - `庫存管理方式`
  - `是否出貨自動扣除`
  - `狀態`
  - `生效日`
  - `失效日`
  - `停用原因`

## 與庫存 / 履約流程的邊界

- 出貨用品屬 Inventory 管理範圍，但不等於每張出貨單都要形成一筆自動耗材扣帳。
- Order / Fulfillment 可以引用用品主資料作為作業指引或包裝標準，但不應直接回寫正式庫存。
- 若未來要升級為半自動扣帳，必須先補充耗用規則、例外處理與 RBAC 邊界，不能直接在現有人工盤點模式上硬切換。

## 與正式扣帳型耗材的邊界

- `提袋加購`、正式包材、內包裝耗材、禮盒姓名貼等若已在主資料與組成 / 耗材規則中被定義為正式扣帳項，就不屬本文件適用範圍。
- 本文件只管理 `2026-03-31_出貨及行政耗材總表_template.csv` 中那批人工盤點型項目。

## 仍待後續擴充

- 倉管 / 履約 / 門市的正式人員層級指派與 row-level 邊界。
- 是否需要區分一次性耗材與可重複使用的包裝資產。
- 若未來升級為半自動扣帳，需另補耗用規則、例外流程與 approval boundary。
