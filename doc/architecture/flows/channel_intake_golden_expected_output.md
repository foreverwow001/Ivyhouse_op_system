# Channel Intake Golden Expected Output

更新日期：2026-03-29

Authoritative source：否（draft）

## 目的

本文件是在 `channel_intake_sample_acceptance_matrix.md` 之下，再往下落一層的逐檔 golden expected output 基線。

本文件的定位：

- 不追求目前就把每一檔的精準筆數全部寫死
- 先定義每一檔最少應出現的 `parsedLine`、`mappingResult`、`exception` 摘要
- 讓後續 parser / mapping / API / 測試開發有逐檔可比對的基線

## 使用原則

### 1. 這是逐檔摘要，不是最終 fixture 明細

- 本版先寫「每檔至少應驗證到什麼」
- 第二版才再往下細化到固定筆數、固定欄位值與固定序列

### 2. `mappingResult` 以正式主資料已補齊為前提定義理想結果

- 若對應商品已在正式主資料與映射規則中存在，預期應映射成功
- 若對應商品尚未建檔，不得靜默略過，應落入 `exception`

### 3. `exception` 只記錄應被保留的例外，不把合法過濾誤列為錯誤

- 例如橘點子 `試吃:` 尾端數量為 `0` 的列，應視為無效需求過濾，不必強制建立 exception
- 但未映射、低信心、規則衝突或新版型，必須可追溯

## 本版尚未寫死的項目

本版故意先不寫死以下內容，避免在未做逐檔標註前產生偽精準：

1. 每檔 `parsedLine` 精準筆數
2. 每檔 `mappingResult` 精準成功筆數
3. 每檔 `exception` 精準數量
4. 每筆 `sourceRowRef` 的完整名單

這些應在下一版 `fixture 級 golden output` 補齊。

## 下一版建議
