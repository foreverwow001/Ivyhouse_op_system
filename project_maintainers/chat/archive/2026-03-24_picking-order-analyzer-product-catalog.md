更新日期：2026-03-24

## 目的

整理 GitHub repo `foreverjojo/picking-order-analyzer` 內建商品資料庫與規格映射，作為後續 Ivyhouse OP System 主資料盤點的外部參考。

資料來源：

- `js/data/products.json`
- `product-mapping.json`
- `README.md`

注意：

- 這份資料較接近「撿貨單分析工具用的標準商品名單 + 報表規格映射」。
- 它不是完整 ERP 等級的商品主檔，未見售價、成本、單位、啟用狀態、版本、有效期間等正式主資料欄位。
- 「禮盒／組合商品」欄位為本次整理時依名稱與分類做的工作判讀，不代表原 repo 有明確欄位標示。

## 初步觀察

1. 這份商品清單已具備集中式標準名稱管理，足夠作為「商品名稱正規化」與「報表欄位映射」來源。
2. 所有列入 `products.json` 的商品，在 `product-mapping.json` 都能找到對應規格映射，代表這份名單與報表模板高度耦合。
3. 禮盒與綜合商品占比不低，若未來移植到正式營運系統，應單獨定義「一般商品」、「組合商品」、「禮盒商品」類型，而不是只靠名稱判讀。
4. 若要把這份資料升級成 Ivyhouse OP System 的正式商品主檔，至少還需要補 SKU / business code、銷售單位、生產單位、是否版本化、是否可售、是否需批次追溯、啟停用規則等欄位。
