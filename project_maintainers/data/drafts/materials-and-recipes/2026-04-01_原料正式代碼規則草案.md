# 原料正式代碼規則草案

## 目的
- 將 72 筆原料草案自暫時轉換清單收斂為可持續擴充的正式 key 規則。
- 避免把品類語意、包裝語意或採購來源直接硬編進 key，降低日後改包裝或改分類時的 shared key 風險。

## 正式 key 規則
- 正式 key 欄位：原料代碼
- 第一版格式：RM0001-RM9999
- 前綴 RM 代表 Raw Material，僅表達主資料族群，不表達品類、供應商、包裝或是否分裝。
- 數字段固定 4 碼、左側補零。
- 第一版已配置範圍：RM0001-RM0072

## 編碼原則
- 一個可治理的單一原料主體只配置一個原料代碼。
- 規格不同且不可互換者應分配不同代碼，例如夏威夷豆 4LL 與 4L。
- 分裝只是作業單位，不另外生成新原料代碼；其單位差異交由單位換算表治理。
- 自製混合料與半成品不得混入 RM 號段，分別維持 MX / SF 草案號段，待正式 owner 定案後再決定是否併入既有 shared key contract。

## 不可變條件
- 原料代碼一旦被 BOM、庫存台帳、工單、扣帳規則或審計資料引用，不得就地改碼。
- 名稱、單位、狀態可調整，但必須保留既有代碼。
- 若需要淘汰代碼，應走停用舊 key + 建新 key + 維護映射的方式處理。

## 第一版配置結果
- 總筆數：72
- 已配置代碼區間：RM0001-RM0072
- 暫未配置語意分段；採來源清單治理順序編碼，避免過早把分類假設寫死進 key。

## 後續建議
- 若原料主檔進入正式 owner 契約，下一版應把本規則升格到 shared_key_contract.md 的獨立 contract。
- 升格前，先完成原料命名規則、單位換算、批次 / 效期策略與 owner 審核責任。
