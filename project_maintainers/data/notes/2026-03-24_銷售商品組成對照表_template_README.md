# 銷售商品組成對照表模板說明

更新日期：2026-03-24

## 檔案用途

這份模板用來建立「銷售商品」與「內包裝完成品」之間的正式對照關係。

適用情境：

- 單一銷售商品直接對應單一內包裝完成品
- 綜合商品由多個內包裝完成品組成
- 禮盒商品由多個內包裝完成品與外包裝材料組成
- 沒有內包裝階段、直接進入可販售狀態的商品

模板檔案：

- `2026-03-24_銷售商品組成對照表_template.csv`

## 本次已預填內容

1. 已納入第一個 tab `商品庫存` 的全部銷售商品與全部販售規格。
2. 每一列代表一個「銷售商品 x 販售規格」組合。
3. 已依目前資料預填建議組成類型。
4. 若銷售商品能直接對應第二個 tab `成品庫存` 的同名內包裝完成品，已預填建議 SKU 與建議單位。

目前總列數：127 列。

## 欄位說明

| 欄位 | 說明 |
| --- | --- |
| 銷售分類 | 第一個 tab 的分類 |
| 銷售商品名稱 | 第一個 tab 的商品名稱 |
| 販售規格 | 第一個 tab 的規格1/規格2/規格3 拆開後的單一規格 |
| 禮盒／組合判定 | 直接沿用第一個 tab 的判定 |
| 建議組成類型 | 系統先行判斷的建議類型 |
| 銷售商品SKU_待建立 | 待你後續建立正式銷售 SKU |
| 組成項次 | 若同一銷售商品規格有多個組成品，請複製同列並改成 1, 2, 3... |
| 對應內包裝成品SKU | 你確認後要正式採用的內包裝完成品 SKU |
| 對應內包裝成品名稱 | 你確認後要正式採用的內包裝完成品名稱 |
| 每1單位銷售商品需用內包裝數量 | 例如一盒雙塔禮盒需要各 5 個塔，就填對應數量 |
| 內包裝扣帳單位 | 例如 個、包、g |
| 是否需要外包裝組裝 | 禮盒與組合商品通常填 `是` |
| 外包裝材料SKU | 若該銷售商品需要盒、袋、貼紙等外包材，可填對應 SKU |
| 外包裝材料名稱 | 外包裝材料名稱 |
| 備註 | 補充規則、組裝說明、例外情況 |
| 對應內包裝成品SKU_建議 | 依現有第二個 tab 自動推得的建議值 |
| 對應內包裝成品名稱_建議 | 依現有第二個 tab 自動推得的建議值 |
| 內包裝單位1_建議 | 第二個 tab 的單位1 |
| 內包裝單位2_建議 | 第二個 tab 的單位2 |

## 建議組成類型說明

| 類型 | 意義 |
| --- | --- |
| `direct_from_inner` | 銷售商品可直接對應一個內包裝完成品 |
| `bundle` | 綜合商品或組合商品，需要多個內包裝完成品 |
| `gift_box` | 禮盒商品，需要多個內包裝完成品，通常也需要外包裝材料 |
| `no_inner_stage` | 沒有內包裝階段，直接就是可販售完成品 |

## 填寫規則

1. 若同一銷售商品規格只對應一個內包裝完成品，就直接在該列填完正式欄位。
2. 若同一銷售商品規格對應多個內包裝完成品，請複製該列，並把 `組成項次` 改成 1、2、3。
3. 禮盒類建議至少拆成：內容物列、外盒列、必要時再加提袋或其他包材列。
4. `對應內包裝成品SKU_建議` 只是預填提示，不代表正式採用。
5. `no_inner_stage` 的商品可以先保留內包裝欄位空白，之後再決定是否要直接以銷售商品本身做扣帳。

## 建議先填的優先順序

1. `雙塔`、`豆塔-綜合`、`堅果塔-綜合`
2. `雪花餅-綜合`、`瑪德蓮-綜合`、`瓦片-綜合`、`西點-綜合`
3. 全部禮盒商品
4. `無調味綜合堅果`

這幾類最需要明確的組成關係，因為它們不能只靠商品名稱推庫存扣帳。
