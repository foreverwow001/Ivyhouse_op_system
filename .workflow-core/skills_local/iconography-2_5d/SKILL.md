# Iconography 2.5D

提供 Ivyhouse Portal 的 2.5D icon 視覺語言，確保 login、landing、流程節點與空狀態使用同一家族 icon，而不是混搭 flat、emoji、outline 或過度卡通化圖像。

## 何時使用

- 建立流程節點 icon
- 建立模組入口卡片 icon
- 建立 login / landing 輔助插圖
- 建立空狀態 icon

## 不應負責的事

- 不定義頁面 layout 與流程映射
- 不處理 API、狀態邏輯與互動回饋

## 核心要求

- icon 必須有輕微體積感與一致透視
- 可愛但不能幼稚，典雅但不能冷硬
- 色彩必須服從 Ivyhouse 品牌 token
- 不得混用多套 icon 語言

## 工作流

1. 先讀 `../../../doc/architecture/modules/portal_brand_token_and_icon_pre_spec.md`。
2. 再讀 [references/icon-style-rules.md](./references/icon-style-rules.md)。
3. 先定 icon family 的角度、光影與 palette，再產出個別 icon。
4. 每批 icon 交付前檢查是否仍屬同一家族。
